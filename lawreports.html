why

A small local law firm contracted me to help with the analytics, operations, product migration, business development, analytics, and even business valuation - honestly, anything that wasn't law, I helped with. In many ways, it felt like working at a startup, given the small size, fast pace, and high ownership.

A key revenue leakage was the firm's loss of billed hours, in which attorneys were sometimes too busy to enter all of their hours worked. The firm's partner tasked me with quantifying and tracking this lost revenue.

To solve this problem, I developed a revenue dashboard, but going beyond the baseline, I automated the entire workflow - from refreshing data pulls via API access to updated modeling to employee-facing reports to sending out emails - to just 2 clicks within on the same page. (No opening email, no copy-pasting of data, no opening of the multiple billing softwares.)


what

automated revenue report (13-tab Excel workbook) features:
  • data pull via API: 1-click button in Excel that automates data pull from Practice Panther into Excel using time-sensitive API access tokens that are refreshed behind the scenes
  • annualized revenue trends: charts that annualize weekly data to identify revenue leakages
  • individualized employee reports: automated emails (via Zapier) that provide each employee with a weekly report of their annualized billed time and productivity trends
user guide (26-page doc) features:
  • how to use automated reports: simple instructions on Excel filepath and to press the buttons
  • adding new employees to report: step-by-step instructions for maintaining automations
  • code: pasted in all code utilized, even shell code used for API token access, for troubleshooting reference
  • zapier set-up: beginner's guide to zapier for troubleshotting purposes
  • legacy reports guide: guide to productivity reports prior to the automation of data pull, where there was VBA code for email templates, but the data had to be manually pasted

how

First, to achieve the tasked goal of quantifying and tracking lost revenue, I quickly developed an Excel model that annualized data, created quotas, included individualized employee tables, and identified revenue losses incurred for estimated lost hours. This was a base report using Excel that did the job, and that I'd email out to each employee weekly, to provide individualized productivity insights.

With that said, this first iteration was heavily manual, and involved too many copy-pastes and too many clicks. I was copy-pasting the data from 3 different systems into Excel, I was copy-pasting relevant tables and attaching the Excels manually into the emails, I was copy-pasting the overall email template each time. Essentially, it was working wonderfully for everyone except me, who was finding it increasingly tedious. The number of steps and context switching to pull data also made it more complex to hand off the task for when my time with the firm ended.

So - I decided to automate the workflow. Using API keys and Zapier, the user only has to click 4 buttons I created on page 1 of the Excel workbook, that will 1) refresh the API timed access token, 2) refresh the data pull, 3) refresh the reports analyses, and 4) pull the new annualized data into employee facing tables, where are then automatically sent through Zapier upon refresh. Initially, I created only 2 buttons - 1) refresh the API timed access token, and 2) refresh the data and automatically seend emails, but expanded it out to four upon request. The main reason was that the director and others at the firm wanted additional visibility into each step in case future troubleshooting needs arose, as they did not feel as comfortable working with code, and wanted simpler step-by-step buttons rather than minimizing 2 clicks. The reasoning here can also be found in the "Suggestions/Advice/Ideas for the future" section of the guide.

In tandem with these reports, I had been heading PMO for our software migration into Practice Panther, a software that provided both timekeeping and billing features. This consolidated our time-entering and billing data into one place, and furthermore, had an API! In other words - the perfect opportunity for automating the data pulls. Utilizing VBA code, Excel's Automate Scripts, PowerQuery, and also a few calls with Practice Panther tech support to activate our API access, I was able to fully automate the data pull into just 2 buttons that the analyst has to press on page 1 of the Excel workbook. I then automated the analytics into 2 more buttons, which annualized the data, then pasted the new analytics accordingly, which triggered a no-touch individualized email report out to employees through Zapier. Again, initially, the Excel data refresh and analytics were all combined into 1 button and the API access refresh being the other button, but the data refresh was separated out into 3 buttons for easy of troubleshooting.

reflections

My first foray into B2B SaaS...? (although this was neither B2B nor SaaS??) On a serious note, this project at work helped me realize how many efficiency gains and consequently profit increases are possible at smaller businesses, particularly as AI is making automation of workflows evermore accessible.

At a personal level, I found this process rewarding. I owned this project from start-to-end, from identifying the opportunity for further automation to researching relevant tools to developing the code with the help of AI to writing the user guide to communicating the necessary change management to analyzing revenue leakage fixes from the reports. Most of all, I was able to make an tangible impact.

If I were to redo this process with my knowledge and experiences now, I can imagine that the automations might be different with the progress of AI. For instance, Zapier seems to becoming more obsolete. Moreover, access to information has always been a key barrier (e.g. knowing how to set up API calls, or knowing names of automation softwares) - that information used bely skills, knowledge, years of effort honed - but now AI can handle much more of the process.

Overall, I found the learning process of this project, and most of all the tangible positive impact, rewarding. At some point in the future, I hope to work again in a smaller-sized company, as I do think the impact possible in and with a small team can be multiples of what is possible elsewhere.
