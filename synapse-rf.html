neural synapse prediction

tldr: trained random forest model to predict neuron synapse formation; key features include nuclei distance, projection group area, and existence of specific proteins such as KIF1A or synaptotagmins

impact: identify trends in synapse formation, where cause-and-effect is not definitively understood



paper (7 pages) →

type: research
status: written oct 2023
tools/stack: python


why

The Baylor Tolias lab collected a dataset of 185k+ Axonal Dendritic Proximity (ADP) instances between neurons in mice, and sought to create a machine learning model that would predict whether an ADP instance would result in a synapse. All neuron synpases form from ADPs, but not all ADPs form synpases. The lab partnered with Rice University's ELEC 478/578 (Electrical Engineering: Machine Learning), and our final project was to create a model that would predict neural synapse formation. See more info on the data set here.

I partnered with Yiming Wang; Yiming primarily owned the research of the existing literature on synpase formation and feature engineering/selection while I primarily owned the machine learning model development/selection/testing and fine-tuning, though we collaborated significantly across our workstreams.


what

methodology:
  • SMOTE (Synthetic Minority Over-sampling Technique): creation of synthetic data so the dataset was more balanced, as less than 1% of ADP instances resulted in synapses in the training data. SMOTE allowed for a more robust training dataset for the ML models
  • feature engineering: engineered features include a feature similarity coefficient, various distance-based features such as nuclei distance, average oracle scores, and a projection group feature capturing the brain location pre- and post- meeting of neurons
  • feature selection: we utilized outside research to support feature selection and engineering, including research which identified importance of neural pathways in synpase formation. ultimately, however, we employed random forest, a machine learning classifier that is relatively less impacted by feature selection
  • model selection: I employed adaboost, random forest, gradient boost, and neural networks to test prediction accuracy. Yiming also employed several logistic models, varying feature selection. Ultimately, we found that random forest performed the best, with adaboost also performing well albeit with limited finetuning.
  • evaluation metrics: we utilized accuracy and balanced accuracy (which better accounts for imbalanced datasets as ours)
    • accuracy: (TP+TN) / (TP+TN+FP+FN) where TP = True Positive, TN = True Negative, FP = False Positive, FN = False Negative
    • balanced accuracy: TP/[2*(TP+FN)] + RN/[2*(TN+FP)] where TP = True Positive, TN = True Negative, FP = False Positive, FN = False Negative

how

The training process was conducted using a pipeline approach in Python’s scikit-learn library. The pipeline encapsulated both data preprocessing and model training steps, ensuring a streamlined workflow. The preprocessing step involved scaling the features using StandardScaler, which standardizes the features by removing the mean and scaling to unit variance. The scaling was particularly important for models like Random Forest that are sensitive to the scale of input features. We divided our provided data into testing and training datasets and analyzed accuracy rates for model selection. Ultimately, we found that random forest was the most robust model, a finding that tracks generally, as random forest tends to be a broadly robust and generalizable classifier.

reflections

This was the most difficult class that I took at Rice, and also one of the most valuable. I especially liked the final project in this class, as it was genuine machine learning research for a real lab's dataset. If I were to make improvements, I would likely have spent more time finetuning the AdaBoost and neural network models, as these tend to be more sensitive to finetuning (whereas a model like random forest does not need as much finetuning due to its robustness). However, the finetuning process was extremelyy time-consuming and memory-consuming for my laptop, which would run for hours each time and start sounding like it was about to explode. Trial by fire for my laptop and I.
